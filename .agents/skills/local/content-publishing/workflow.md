# Content Publishing Workflow

1. Review updates to `src/pages`, `src/features`, `src/hooks`, and `src/services`.
2. Validate route transitions and base-path handling.
3. Verify fetch/error/loading states for control-plane APIs.
4. Run `npm run lint` and `npm run build`.
5. Smoke test in mock mode and control-plane mode if integration paths changed.
6. Capture screenshots or notes for any UX-affecting change.
